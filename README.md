# shijiecup-lab

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库用于存放各类独立 HTML 页面，涵盖工具、演示、实验性页面等。所有页面均为静态 HTML 文件，可直接在浏览器中打开或通过 GitHub Pages 进行访问。

## 目录结构

```
shijiecup-lab/
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库主页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，便于管理和引用。
- `assets/`：存放多个页面共用的样式、脚本或媒体文件。

## 页面归档说明

- 每个页面均为独立的 HTML 文件，不依赖外部数据库或后端服务。
- 页面内容不针对任何特定域名或网站，仅作为功能演示或实验性存档。
- 部分页面可能包含交互或动态效果，均使用纯前端技术实现。
- 页面命名应尽量简洁，建议使用英文或拼音，避免特殊字符。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能将根目录或 `docs/` 目录发布为静态站点。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来改进现有页面或添加新页面。
- 新增页面请放置在 `pages/` 目录下，并尽量保持目录结构清晰。
- 若页面使用了公共资源，请确保资源路径正确，避免引用失效。
- 本仓库不追踪具体使用场景，仅负责页面内容的归档与发布。
